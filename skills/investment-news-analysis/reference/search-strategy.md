# 搜索策略详解

## 核心原则

搜索的目标不是堆信息，而是为当日 summary 和建议页面提供可验证证据。

搜索顺序先按“政策主线怎么走、市场是否确认、宏观是否顺风”来建立当天框架，再决定是否补基金本身或同类比较细节。

每次完整分析时，搜索结果必须足够支撑：

1. 当日 summary 第二章和第三章
2. 短期预测的信息充分性检查
3. 逐基金状态复核与操作建议
4. 政策新方向雷达

## 搜索顺序

### 第一层：先读今天本地 `raw_data/finance_news.json`

先读：

1. 今天目录下的 `raw_data/finance_news.json`
2. `index.json`
3. 近期 summary
4. 最新 `持仓情况.md`

目的：先利用今天已经抓到的财经快讯建立当天素材池，再结合历史判断和当前缺口决定后续搜索。

读取今天的 `finance_news.json` 时，至少先做三件事：

1. 按政策、市场行情、宏观三个桶重新分组。
2. 标记哪些主题已经有足够证据，哪些仍有缺口。
3. 把与国内市场经济和持仓完全无关的噪音先剔除。
4. 筛选出有用的新闻之后拉取原文，总结归档到本地。

注意： 后续所有搜索都是对本轮筛选后产物的补充。

### 第二层：再搜最近一周的政策面

再搜会改变主线判断的政策信息。

搜索：

1. 与持仓主题直接相关的政策文件
2. 部委、国务院、行业监管的新表态
3. 产业扶持、限制、整治、补贴、扩容等政策动作
4. 未来 3-6 个月可能受政策支持的新方向

目的：先确认当天应该围绕什么主线理解市场，而不是先陷入零散个基新闻。

### 第三层：再搜最近1天市场行情面，只看今天和昨天的

政策之后，立即确认市场是否已经给出价格和量级反馈。

搜索：

1. 指数、行业 ETF、北向方向
2. 成交额、换手、风险偏好、市场量级变化
3. 相关 ETF 近 1 月申赎或资金流状态
4. 与持仓映射的主线行情强弱

这一层必须覆盖：

1. 近 4 周日均成交额与过去 3 月日均的对比。
2. 北向近 1 周方向。
3. 前一交易日 ETF 收盘强弱排序。

缺少这三项中的任一项，动量判断必须降级。

### 第四层：最后搜最近一周的宏观面

在政策主线和市场确认之后，再补宏观背景，判断顺风还是逆风。

搜索：

1. 国务院 / 部委政策
2. 流动性与宏观数据
3. 经济周期位置
4. 海外宏观联动

目的：宏观在这里主要起解释和校正作用，不应抢在政策和市场前面定义当天主结论。

### 第五层：基金本身信息补缺

完成前三层主搜索后，再回头补与持仓直接相关的个基信息。

这一层是低优先级补充，不是当天主链路。

搜索：

1. 基金公告
2. 基金经理变化
3. 基金公司表态
4. 季报 / 定期报告

如果基金本身没有新增直接新闻，不记录、不归档，直接跳过此层。

### 第六层：同类基金横向比较

目标：

1. 找到同类可比基金近期表现
2. 确认当前持仓是强、弱还是中性
3. 为横向比较和动作建议服务

### 第七层：政策新方向雷达

搜索未来 3-6 个月可能受政策支持、且存在场外基金承载的方向。

输出只用于观察，不直接跳成建仓建议。

## 主搜索链路的判断逻辑

当天的默认顺序固定为：

1. 先读今天 `raw_data/finance_news.json`
2. 先搜政策面
3. 再搜市场行情面和量级新闻
4. 最后搜宏观面

只有在这四步跑完后，才进入基金本身补缺和同类横向比较。

## 搜索结果判断标准

保留的信息应该能回答至少一个问题：

1. 这条信息会改变当前动作吗？
2. 这条信息会改变风险判断吗？
3. 这条信息能补齐当前缺口吗？

如果都不能，通常不必归档。

## 搜索关键词原则

1. 先用 `finance_news.json` 里的标题和主题词做当天第一轮关键词池。
2. 政策类优先用“政策主题 + 部门 / 文件类型”。
3. 市场行情类优先用“指数 / ETF / 北向 / 成交额 / 风险偏好 + 最近”。
4. 宏观类优先找直接可传导的流动性、增长和海外联动信号。
5. 基金本身补缺时再用“基金名称 + 代码”。

## 搜索后的立即动作

每轮搜索后都要做两件事：

1. 归档高/中关联信息。
2. 判断当前还缺哪一类证据。

## 不再作为默认目标的旧搜索产物

以下内容不再作为日报默认主结构的目标字段：

1. 旧版情绪总表
2. 第三方观点汇总章节
3. 个股层逐条跟踪骨架

如果实际搜索到高价值相关信息，可作为事实依据写入日报，但不再单独强制产出这些章节。
