# 归档流程详解

## 核心原则

1. 每轮搜索后确认目标新闻是否已经归档到本地，若有则忽略。
2. 若本地没有归档则立即进行归档，不要等到最后统一保存。
3. 只保存高关联和中关联信息。
4. 外部搜索信息进入 `item_summaries/`，脚本输出才进入 `raw_data/`。


## 归档顺序

### 步骤 1：执行搜索

搜索对象包括：

1. 政策面信息
2. 市场行情与量级信息
3. 宏观面信息
4. 基金本身公告 / 基金经理 / 基金公司信息（低优先级补缺）

### 步骤 2：做关联性判断

评分建议：

- 高关联：80-100，必须保存
- 中关联：50-79，建议保存
- 低关联：0-49，不保存

评分维度：

1. 是否直接影响当前持仓
2. 是否来自正式或权威来源
3. 是否足够新
4. 是否能改变当前判断、风险边界或动作优先级

### 步骤 3：生成单条摘要

每条高/中关联信息都生成独立 Markdown 文件。

推荐字段：

```markdown
### [YYYY-MM-DD] 事件标题

**核心要点**：
**影响判断**：✅ 利好 / ⚠️ 中性 / ❌ 利空
**相关持仓**：基金名称(代码)
**信息来源**：[来源名](URL)
**抓取时间**：YYYY-MM-DD HH:MM:SS
```


### 步骤 4：保存 raw_data

只有脚本产出的结构化文件进入 `raw_data/`。

外部搜索到的新闻、政策、市场和宏观信息，不额外存 JSON，统一通过 `item_summaries/` 归档。

常见文件：

```text
finance_news.json
market_momentum_YYYY-MM-DD.json
analysis_snapshot_YYYY-MM-DD.json
```

### 步骤 5：生成每日 summary

所有搜索完成后，再读取当日 `item_summaries/` 和 `raw_data/` 生成 `summary_YYYY-MM-DD.md`。

日报结构以 [daily-summary-template.md](daily-summary-template.md) 为准。

## 目录示意

```text
投资新闻归档/YYYY-MM/YYYY-MM-DD/
├── item_summaries/
├── raw_data/
└── summary_YYYY-MM-DD.md
```

## 去重规则

1. 标题高度相似且来源相同，不重复归档。
2. 同一事件的多来源报道，可保留一个主来源并在单条摘要中注明补充来源。
3. 如果本地已存在等价摘要，不再创建第二份近似文件。

## 最低归档线

1. 8 持仓完整日报默认不少于 8 条单条摘要。
2. 普通完整日报默认不少于 6 条单条摘要。
3. `raw_data/` 中必须存在可溯源结构化 JSON。
4. `summary` 中必须写 `item_summaries_count` 和主要 raw_data 文件名。

## 常见错误

1. 等全部搜索完再统一保存。
2. 只保存 raw_data，不写单条摘要。
3. 只写利好，不补反向证据。
4. 没有任何基金直接新闻时，直接跳过归档。
