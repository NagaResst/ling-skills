# 预测区间调整与数据降级参考

## 预测区间宽度调整规则

### 问题
连续同方向大幅波动日（如连续两日+4%+或连续两日-3%+）后，上一轮预测区间上限极易被突破。
实证：2026-06-30 准确率从 90.9% 降至 54.5%，5个错误中3个为"微超上限"（0.003~0.005），2个为"超幅较大"（0.109~0.51）。

### 规则
1. **正常市场**（前一交易日波动 < 3%）：使用标准区间宽度。
2. **连续同方向波动**（前一交易日 ≥ 3% 且前两日同方向 ≥ 3%）：高波动基金（浮盈 > 15% 或日变动 > 3%）的预测区间宽度扩大 30-50%。
3. **单日极端波动**（前一交易日 ≥ 5%）：所有权益类基金预测区间宽度扩大 20-30%。
4. **方向判断不变**：扩大的是区间宽度，不是改变方向。方向仍由政策面+基本面+北向趋势决定。

### 实操
- 债券类基金：区间通常 0.02 宽，极端日扩大到 0.03。
- 宽基增强类：区间通常 0.07 宽，极端日扩大到 0.10。
- 高波动权益类（芯片/科创/主动科技）：区间通常 0.20-0.30 宽，连续反弹日扩大到 0.30-0.45。
- 限购慢加类：区间通常 0.15-0.20 宽，极端日扩大到 0.20-0.30。

### 验证口径
- "微超上限"（< 0.01）属于区间设定偏窄，不算方向错误。
- "超幅较大"（> 0.05）反映市场力度超出预期，需在复盘结论中标注。
- 准确率低于 60% 时，下一轮必须全面拓宽区间。

## 申万二级行业数据降级

### 检测
```python
if mom['sw_l2_industry_daily']['status'] == 'empty':
    # 进入降级模式
```

### 降级方案
1. 以 `relevant_etf_daily` 中持仓相关ETF涨跌作为板块代理。
2. 以 `core_industry_etf_daily` 中宽基ETF作为市场风格参考。
3. 日报第四章用"持仓相关行业代理"表替代申万二级行业涨跌幅排行。
4. 数据附录标注缺口。

### 不阻塞
申万数据缺失不构成预测阻塞项。ETF代理足以支撑：
- 板块方向判断（涨/跌/平）
- 持仓基金相对强弱排序
- 行业轮动信号（如科技→医药轮动可通过ETF代理观察）

## 清仓基金的处理

### 快照检测
`analysis_snapshot` 的 `holdings_change_vs_previous_report.changes[]` 中 `change_type: "cleared"` 的基金即为已清仓。

### 报告处理
1. 不进入当前活跃持仓列表、目录、卡片、摘要表、fund_cards_json。
2. 在"持仓变化检测"中写明清仓金额和盈亏。
3. ETF总览表中保留对应行但标注"已清仓"。
4. 如果被清仓基金的ETF代理仍在 `relevant_etf_daily` 中，保留行但改备注。
