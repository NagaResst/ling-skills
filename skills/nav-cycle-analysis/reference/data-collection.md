# 净值数据采集规范

## 🎯 目标

建立可持续更新的本地净值时序数据库，支撑周期分析和预测。

---

## 一、数据来源与优先级

| 优先级 | 来源 | 说明 | 适用场景 |
|-------|------|------|---------|
| 1 | 天天基金网（eastmoney） | 数据完整，最早可追溯10年+ | 主力来源 |
| 2 | 蛋卷基金（danjuanfunds） | 界面友好，支持导出 | 辅助验证 |
| 3 | 基金公司官网 | 最权威，但格式不统一 | 数据质量核验 |
| 4 | 天天基金历史净值API | 非官方但可用 | 批量采集 |

**天天基金历史净值 URL 格式**：
```
https://fund.eastmoney.com/f10/F10DataApi.aspx?type=lsjz&code={基金代码}&page=1&sdate=2020-01-01&edate=2026-12-31&per=40
```

## 三、数据完整性检查

每次加载数据后必须执行以下检查：

| 检查项 | 通过标准 | 失败处理 |
|-------|---------|---------|
| 记录数量 | ≥ 50个交易周（约250条日数据）| 标注置信度降级，分析继续 |
| 日期连续性 | 无超过5个交易日的断层 | 标注数据缺口，不插值 |
| 净值合理性 | 无单日涨跌幅超过±20%的异常值 | 标注并人工复核 |
| 基金经理一致性 | 数据范围须与当前经理任职时间取交集 | 截断至任职起始日 |

---

## 四、更新频率

| 场景 | 建议更新频率 | 说明 |
|------|------------|------|
| 周期分析（常规） | 每周一次 | 取上周最新净值追加 |
| 急跌/急涨时 | 每日 | 净值单日变化 > ±3% 时触发 |
| 政策事件发生后 | 事件后连续5个交易日 | 追踪事件冲击的完整衰减过程 |

---

## 五、本地存储格式

**文件路径**：`净值周期分析/{基金代码}/nav_history.json`

**追加原则**：每次只追加新日期的数据，不覆盖历史。如本地已有至 2026-05-08，则只追加 2026-05-09 及之后的数据。

**数据量估算**：
- 5年日数据约 1200条，JSON 文件约 80–120KB
- 每周追加 5条，增量极小，无需担心存储

---

## 六、置信度分级

| 数据量 | 置信度等级 | 分析限制 |
|-------|----------|---------|
| < 50交易周 | 🔴 低 | 仅可做趋势判断，不可识别完整周期 |
| 50–150交易周 | 🟡 中 | 可识别1–2个完整周期，统计规律不稳定 |
| 150–250交易周 | 🟢 高 | 可识别3个以上完整周期，统计规律较可靠 |
| > 250交易周 | 🟢 高+ | 可跨两个五年规划周期分析结构性变化 |
