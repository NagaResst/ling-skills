# 单持仓研究规范

## 核心定位

本规范服务于**以全局持仓诊断与建议为前置的单个当前持仓研究**。

它不是报告模板，也不是示例集合，而是一份统一的研究规范，负责同时约束：

1. 研究顺序。
2. 必读输入。
3. 必答问题。
4. 输出结构。
5. 核心建议写法。

使用本规范时，先回答组合位置，再回答单基金动作，不得跳过组合层直接下单基金结论。

---

## 适用范围

适用于：

1. 验证一只当前已持仓基金的原持仓逻辑是否仍成立。
2. 判断这只基金在当前组合里是否还适合现在这个位置和仓位。
3. 输出继续持有 / 不加仓 / 加仓 / 减仓 / 退出等动作建议。
4. 生成单持仓分析报告。

不适用于：

1. 新基金首次研究。
2. 不依赖组合上下文的独立单基金深度研究。

---

## 输入优先级

研究前至少读取以下材料，且顺序不能乱：

1. `投资者行动/投资者画像.md`
2. `投资者行动/持仓情况.md`
3. `投资新闻归档/YYYY-MM/YYYY-MM-DD/raw_data/market_momentum_YYYY-MM-DD.json`
4. `投资新闻归档/YYYY-MM/YYYY-MM-DD/raw_data/analysis_snapshot_YYYY-MM-DD.json`
5. 同一分析日其他必要 `raw_data/*.json`

硬规则：

1. 组合层结论必须由 `投资者行动/投资者画像.md` 里的目标结构和当日 `raw_data` 元数据共同推导，不得直接复用过去生成的日报或建议。
2. 如果目标基金不在当前持仓中，不继续执行本规范。
3. 如果当日元数据没有覆盖目标基金，必须明确写数据缺口，并降低建议强度。
4. 如果组合结论和单基金结论冲突，必须解释冲突来自哪条证据或哪条数据缺口。
5. 如果画像里已经给出“建议的投资组合结构”或等价目标配比，仓位偏重 / 偏轻判断必须先对照这套目标结构，不能只按当前自然权重下结论。

## 研究与建议的依据

本规范的“研究”和“建议”不是凭主观感觉下结论，而是建立在可复核的投资实务框架上。这里的“科学依据”不是指单一万能模型，而是指一套可重复、可解释、可被反证推翻的证据体系。

### 1. 组合管理依据

先做组合位置校准，依据的是组合管理里的角色分工与风险预算原则：

1. 同一只基金的好坏，必须放回组合里看它承担的是稳定器、核心宽基、主题仓还是观察仓。
2. 同方向暴露过多会抬高组合波动和回撤，所以“是否适合当前仓位”比“基金本身是否还可以拿”更重要。
3. 调仓建议优先处理的是角色错配、仓位失衡和主题重叠，而不只是短期涨跌。

### 2. 相对强弱依据

使用近 1 月表现、相对代理基准超额、近 30 天高点回撤，不是为了机械追涨杀跌，而是为了判断趋势延续性和相对效率：

1. 相对代理基准超额用于判断这只基金是否跑赢它本该代表的市场段。
2. 回撤幅度用于判断修复质量和风险暴露，而不是只看绝对涨跌。
3. 如果一只基金持续弱于同类基准或主线 ETF，继续提高仓位的依据就会明显变弱。

### 3. 多源确认依据

本规范要求同时看官方净值、ETF / 指数、市场量级、基金自身公告，依据的是多源确认原则：

1. 单一来源的强弱信号容易失真，所以不能只看一日净值或一条新闻就下强结论。
2. 官方净值提供已确认结果，ETF / 指数提供更高频的主线温度，公告与基金经理信息提供基本面变化线索。
3. 只有多条证据互相印证时，才支持更强动作；证据不齐时，默认降低建议强度。

### 4. 决策与风险控制依据

动作集合优先出现“继续持有但不加仓”和“暂不动作，等待确认”，依据的是决策理论中的错误成本不对称：

1. 在证据不足时贸然加仓，代价通常大于先等待确认。
2. 对已持有基金，错误地把弱修复写成强趋势，会放大回撤风险。
3. 因此，本规范默认先控制误判成本，再争取收益弹性。

### 5. 行为偏差校正依据

本规范要求先看当日元数据、再区分事实与推导、单列信息边界，依据的是对常见行为偏差的约束：

1. 防止锚定在买入成本上，把“回本”误当成研究依据。
2. 防止沉没成本效应让弱逻辑被无限拖延。
3. 防止只看支持证据，不看反证和数据缺口。

### 6. 证据层级原则

研究时默认按以下顺序给证据分层：

1. 已确认数据：官方净值、持仓份额、已归档公告、已归档 raw_data。
2. 当日高相关市场证据：ETF / 指数强弱、市场量级、北向等。
3. 当日基金相关结构化证据：同一分析日 `raw_data` 中的基金事实、公告线索、缺口说明。
4. 推导性判断：对后续动作的条件化建议。

低层级证据不能反过来覆盖高层级证据；推导性判断必须建立在前面三层之上。

### 7. 使用边界

这套依据支持的是“更稳健地做持仓管理”，不是保证收益，也不是证明某只基金一定会上涨。

因此：

1. 本规范能提高建议的一致性和可解释性，但不能消除市场不确定性。
2. 当关键数据缺失时，本规范的正确做法不是补猜测，而是降低建议强度并明确边界。

## 强制研究顺序

## 五类必答信号

为避免研究只剩流程而失去判断抓手，单持仓分析时必须显式回答以下五类信号。这五类信号是本规范里的最小判断骨架。

### 1. 组合位置是否仍合适

回答：这只基金在当前组合里还应不应该占这个位置和这个仓位。

重点看：

- 投资者画像里的建议组合把现金、防御、核心 Beta、政策主线分别放在什么目标区间。
- 当前组合总诊断是防守偏厚、核心偏薄，还是主题重叠偏高。
- 这只基金承担的是稳定器、核心宽基、主题仓还是观察仓。
- 组合层结论更接近继续保留、调仓，还是更换候选。

### 2. 原逻辑是否还在

回答：最初持有它的主线还在不在。

重点看：

- 原正证是否还成立。
- 原风险是否扩大。
- 是否出现新的反证。

### 3. 当前强弱是否支持继续拿

回答：这只基金相对代理基准、相对同主题 ETF、相对组合内其他持仓是强还是弱。

重点看：

- 近 1 月超额收益。
- 回撤后修复力度。
- 是否持续弱于主线 ETF。

### 4. 确认级别是否足够

回答：当前判断是否被官方净值、ETF、北向继续确认。

重点看：

- 是否只有单日修复。
- 是否缺少连续确认。
- 是否存在关键数据缺口。

### 5. 信息边界在哪里

回答：这次为什么不能写得更激进。

常见边界：

- 基金本身没有新增直接催化。
- 北向或量级证据不够强。
- 官方净值尚未出现更连续确认。

---

### 阶段一：先做组合位置校准

先用 `投资者行动/投资者画像.md` 里的建议组合，再结合当日 `market_momentum` 和 `analysis_snapshot`，回答这只基金在当前组合里该不该占现在这个位置和仓位。

最低要回答：

1. 画像里的目标结构是什么。
2. 当前组合总诊断是什么。
3. 这只基金在组合中的角色是什么。
4. 当前仓位是否符合组合建议。
5. 在组合层更接近继续持有、调仓，还是更换候选。

常见写法：

- 画像目标结构：现金 / 防御 / 核心 Beta / 政策主线各自目标占比为多少。
- 当前组合总诊断：当前组合防守偏厚、宽基偏薄、主题内部重叠偏高。
- 这只基金在组合中的角色：观察仓 / 稳定器 / 核心宽基 / 主题仓。
- 当前仓位是否符合组合建议：符合 / 略重 / 偏轻 / 仅适合观察仓。
- 在组合层更接近：结构保留 / 控制仓位 / 优先复核 / 更换候选。

### 阶段二：再锁定当前持仓快照

至少确认：

1. 当前份额。
2. 当前持仓成本。
3. 前一交易日官方净值。
4. 当前浮盈浮亏。
5. 近 1 月表现。
6. 相对代理基准超额。
7. 从近 30 天高点回撤。

### 阶段三：整理当日元数据证据链

优先从当日 `raw_data` 元数据中整理：

1. 官方净值与持仓金额变化。
2. 相对代理基准超额与回撤质量。
3. ETF / 指数和市场量级确认。
4. 当日基金相关新增事实与关键缺口。

### 阶段四：校验当前逻辑是否仍成立

必须逐条回答：

1. 原主线是否还在。
2. 当前强弱是否支持继续拿。
3. 市场与 ETF 是否继续确认。
4. 基金本身是否有新增催化或新增风险。
5. 当前最关键的反证是什么。

### 阶段五：输出动作建议

动作只能落在以下集合之一：

1. 继续持有。
2. 继续持有但不加仓。
3. 逢回撤分批加仓。
4. 先减仓一档。
5. 退出观察仓 / 清仓。
6. 暂不动作，等待确认。

## 输出结构规范

最终报告建议按以下结构组织，但这里强调的是结构职责，不是机械填空模板：

1. 当前持仓快照
2. 组合位置校准
3. 原持仓逻辑回放
4. 当前逻辑是否仍成立
5. 核心建议
6. 具体调仓方案
7. 下次确认点

最低要求：

1. 第二部分必须先于单基金动作结论出现。
2. 第五部分必须给出完整五项建议语法。
3. 若仓位已经很轻，优先回答观察仓是否继续存在。
4. 若没有新增基金公告，不要硬补结论，把缺口写进信息边界。

---

## 核心建议固定语法

核心建议必须统一写成：

```markdown
- 当前建议：
- 什么时候动作：
- 判断理由：
- 事实依据：
- 信息边界：
```

要求：

1. `当前建议` 只能写一个主动作。
2. `什么时候动作` 必须给出触发条件，或明确写继续观察。
3. `判断理由` 解释动作方向，不堆数据。
4. `事实依据` 只写已确认数据与已归档信息。
5. `信息边界` 必须交代为什么不把结论写得更强。

## 常见写法提醒

1. 没有足够确认时，优先写“继续持有但不加仓”。
2. 仓位已经很轻时，优先回答“观察仓是否继续存在”。
3. 没有基金新增公告时，不要硬补结论，把缺口写进“信息边界”。

---

## 可选增强项

当单持仓争议较大、仓位较重，或组合层结论与单基金层结论存在明显张力时，可以追加：

1. 情景分析。
2. 仓位过重风险检查。
3. 分批加减仓路径。
4. 当日元数据缺口敏感性说明。

这些增强项只用于强化判断，不改变前面的强制研究顺序。

## 最小写法示例

以下片段只展示规范内的推荐写法，不构成独立模板：

```markdown
## 组合位置校准

- 当前组合总诊断：当前组合防守偏厚、宽基偏薄、主题内部重叠偏高。
- 这只基金在组合中的角色：观察仓 / 可选弹性仓，不承担核心仓职责。
- 当前仓位是否符合组合建议：符合。当前仓位已经足够轻，不需要为它挤占核心仓或稳定器仓位。
- 在组合层更接近：结构保留，继续观察，不是优先更换产品对象。

## 核心建议

- 当前建议：继续持有但不加仓。
- 什么时候动作：只有在官方净值与主题 ETF 连续确认后，再考虑恢复加仓讨论。
- 判断理由：当前逻辑未失效，但强度不足以支持更激进动作。
- 事实依据：前一交易日官方净值、组合横向强弱、主题 ETF 日线、当日 `market_momentum` 与 `analysis_snapshot` 元数据。
- 信息边界：基金本身新增直接催化不足，当前更像市场带动修复而不是基金独立转强。
```

---

## 交付前检查

交付前至少确认：

1. 是否先写了组合位置校准，再写单基金动作。
2. 是否明确回答了当前仓位是否符合组合建议。
3. 是否只使用了当日 `raw_data` 元数据，而没有引用过去的日报和建议。
4. 核心建议五项是否齐全。
5. 是否把未确认信息留在信息边界，而不是混进主结论。
6. 是否明确写出下次确认点。
