# 政策事件映射规范

## 🎯 目标

建立「政策/市场重大事件 ↔ 净值变化」的因果关系库，量化不同类型事件对净值的平均影响幅度和滞后时间，用于预测模型中的「政策信号」维度。

---

## 一、事件分类体系

### 一级分类

| 分类代码 | 分类名称 | 说明 |
|---------|---------|------|
| `POLICY_POS` | 政策利好 | 产业补贴、税收优惠、国家级发展规划 |
| `POLICY_NEG` | 政策利空 | 监管收紧、行业整治、补贴退坡 |
| `MACRO_EAS` | 宏观宽松 | 降准、降息、流动性注入 |
| `MACRO_TIG` | 宏观收紧 | 加息、缩表、资金面收紧 |
| `INDUSTRY_POS` | 行业景气利好 | 出货量创新高、PMI回升、景气指数好转 |
| `INDUSTRY_NEG` | 行业景气利空 | 需求下滑、库存高企、价格战 |
| `GEOPOLITICAL` | 地缘政治 | 关税、制裁、贸易摩擦 |
| `HOLDING_EVENT` | 重仓股事件 | 重仓股业绩、管理层、并购 |

## 二、事件库存储格式

**文件路径**：`净值周期分析/{基金代码}/policy_events.json`

```json
{
  "fund_code": "003984",
  "fund_name": "嘉实新能源新材料A",
  "events": [
    {
      "event_id": "E001",
      "date": "2022-06-01",
      "category": "POLICY_POS",
      "title": "国务院发布新能源汽车产业发展规划（2021-2035）",
      "description": "明确2025年新能源汽车销量达到20%，延续购置税减免",
      "impact_observed": {
        "1w_return": 0.082,
        "2w_return": 0.143,
        "4w_return": 0.198,
        "8w_return": 0.156
      },
      "lag_weeks": 1,
      "confidence": "high",
      "source_url": "https://www.gov.cn/..."
    }
  ]
}
```

**字段说明**：
- `lag_weeks`：净值实际反应的滞后时间（周）
- `impact_observed`：事件后各节点的净值变化幅度（相对事件日）
- `confidence`：该事件影响的确认程度（high/medium/low）

## 三、标准事件类型影响参考值

> 以下为跨多只同类基金的统计参考值，实际值因基金差异有较大波动，使用时须结合具体事件库数据校正。

| 事件类型 | 平均净值反应（4周） | 典型滞后 | 持续时间 | 注意事项 |
|---------|---------------|---------|---------|---------|
| 国家级产业政策（强力） | +10%~+20% | 0–1周 | 2–4个月 | "规划"类持续时间更长；"补贴"类有退坡风险 |
| 国家级产业政策（普通） | +3%~+8% | 1–2周 | 1–2个月 | 多为配套文件，跟随主政策走 |
| 行业监管收紧 | -15%~-35% | 0周（立即） | 1–6个月 | 影响程度取决于监管覆盖范围 |
| 补贴退坡公告 | -5%~-15% | 0–1周 | 1–2个月 | 已充分预期的退坡影响相对小 |
| 全面降准/降息 | +3%~+8% | 2–3周 | 持续至政策周期结束 | 对纯债基金影响方向相反 |
| 景气数据超预期（月度） | +5%~+12% | 0–1周 | 2–4周 | 需连续超预期才能持续推高净值 |
| 景气数据低于预期 | -3%~-10% | 0–1周 | 2–4周 | 单次miss影响有限 |
| 关税超预期上调 | -10%~-25% | 0周 | 2–8周 | 出口相关板块影响更大 |
| 重仓股业绩大幅低于预期 | -5%~-15% | 0周 | 1–3周 | 集中度越高影响越大 |
| 重仓股业绩超预期 | +3%~+8% | 0–1周 | 2–4周 | — |

## 四、事件采集流程

### 4.1 新增事件时的操作步骤

1. **搜索**：使用 `investment-news-analysis` 的搜索策略中的「行业政策」层，获取近期政策动态
2. **分类**：按上表一级分类进行归类
3. **记录起始净值**：记录事件日当天的净值作为 baseline
4. **追踪4–8周**：在后续报告中补充 `impact_observed` 的各节点数据
5. **写入 JSON**：追加到 policy_events.json

### 4.2 初次建立事件库时（回溯历史）

1. 整理近3年的重大政策事件时间线（搜索关键词：「[行业名称] 政策 年份」）
2. 对照净值历史，补充 `impact_observed` 数据
3. 每次完成后统计各事件类型的平均影响值，校正上表参考值

## 五、政策日历（前瞻）

在每份周期报告中，必须列出未来4–8周内已知的高影响力政策节点：

```markdown
## 政策日历（未来8周）

| 预计日期 | 事件 | 类型 | 历史影响参考 | 影响方向预判 |
|---------|------|------|------------|------------|
| 2026-05-15 | 4月新能源汽车销量数据发布 | INDUSTRY数据 | 好于预期+8%；差于预期-5% | 待数据 |
| 2026-05-下旬 | 国常会可能讨论消费刺激政策 | POLICY_POS潜在 | 正面影响 | 待确认 |
| 2026-06-初 | 美联储6月会议纪要 | MACRO宏观 | 鹰派-3%；鸽派+2% | 待数据 |
| 2026-06-30 | 新能源汽车补贴政策到期节点 | POLICY关键节点 | 若续期+5%；若退坡-10% | 待确认 |
```

**使用规则**：
- 只列出已确定日期或高概率发生的事件
- 不确定的事件标注「待确认」，不得捏造发布时间
- 每次更新周期报告时同步更新此日历
