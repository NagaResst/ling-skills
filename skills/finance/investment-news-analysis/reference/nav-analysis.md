# 净值分析规范

## 核心定位

净值分析在本 skill 中只承担两个职责：

1. 量化当前持仓所处的位置。
2. 为动作判断提供辅助边界和触发线参考。

它不负责输出单基金深度技术分析长文，也不单独决定建仓与否。

## 必看指标

### 1. 前一交易日官方净值

这是日报与建议页面里唯一可直接引用的基金净值口径。

### 2. 浮盈浮亏

```text
浮盈率 = (当前官方净值 - 持仓成本) / 持仓成本
```

作用：

1. 判断观察仓是否仍弱。
2. 判断强势仓是否接近止盈纪律。

补充：止盈线、减仓线要先锚定持仓成本，再写成绝对价格；如果成本没变，就更新“离既有止盈线还有多远”，不要因为最新净值上涨而把线一起抬高。只有成本变化或策略重估时，才改线。

### 3. 近 1 月表现

作用：

1. 确认近期强弱方向。
2. 服务横向比较。

### 4. 相对代理基准超额

作用：

1. 判断基金是否跑赢对应宽基或主题 ETF。
2. 识别“自身在涨，但其实弱于主线”的情况。

### 5. 从近 30 天高点回撤

作用：

1. 判断修复后是否已接近高位。
2. 判断是否仍在回撤中。

### 6. 历史分位（可选增强）

可以作为辅助参考，但不作为本 skill 的单独主结论来源。

如果使用，必须说明：

1. 采用的数据窗口
2. 数据覆盖是否完整
3. 该分位只作为辅助，不直接等于建仓建议

## 推荐时间窗口

1. 近 30 天：动作判断主窗口
2. 近 90 天：回撤和修复辅助窗口
3. 更长窗口：仅在需要辅助说明历史位置时使用

## 在动作判断中的用法

### 观察仓

重点看：

1. 是否仍低于成本
2. 是否持续弱于宽基 / 主线 ETF
3. 是否跌破观察线

### 底仓

重点看：

1. 是否仍在中性震荡带内
2. 是否继续适合持有不追买

### 强修复仓

重点看：

1. 是否继续跑赢宽基
2. 是否已接近止盈线
3. 是否只是一天强，还是被后续确认

### 稳定器

重点看：

1. 是否保持低波动
2. 是否继续承担稳定功能

## 简单示例

```python
def floating_pnl(nav, cost):
    return round((nav - cost) / cost * 100, 2)

def excess_return(fund_return, benchmark_return):
    return round(fund_return - benchmark_return, 2)

def drawdown_from_high(current_nav, recent_high):
    return round((current_nav - recent_high) / recent_high * 100, 2)
```

## 使用边界

1. 净值指标只能基于确认数据。
2. 不能把单一净值信号直接上升为重动作建议。
3. 不能脱离 ETF、北向、政策与基金本身信息单独下结论。
4. 净值分析永远要回到“这只基金今天该怎么处理”上，而不是展开成长篇技术分析。
