# 预测验证机制

## 核心目标

预测验证的职责只有两件事：

1. 验证更早 summary 里的短期预测是否被实际净值验证。
2. 基于验证结果，修正当前一轮预测的边界和置信度。

本文件只约束日报中的：

- `历史预测验证`
- `短期预测（1-4周）`

## 规则零：预测前必须通过信息充分性检查

任何方向性预测都必须先确认信息足够，而不是为了给答案强行写结论。

最低要求：

1. 已有基金自身行为数据：前一交易日官方净值、近 1 月表现、相对基准超额、近期回撤或浮盈浮亏。
2. 已有市场环境数据：成交活跃度、北向或市场量级、相关 ETF 资金流或申赎基线。
3. 已有行业 / 政策 / 产业证据：至少 1 条正式信息支撑主线判断。
4. 已有反向证据、风险证据或明确数据缺口。
5. 已完成至少 1 组横向比较：与代理基准或同类基金比较强弱。

判定规则：

- 前三项缺任一项：不得输出方向性预测。
- 第四项缺失：视为只报喜不报忧，报告失效。
- 第五项缺失：不得输出高置信度预测。

补充说明：

- 基金本身新增公告 / 经理变更信息有则补充，没有也不构成阻塞条件。

## 规则一：每日日报必须包含独立 `短期预测（1-4周）` 章节

短期预测必须是独立二级标题，方便后续提取验证。

不要求放在全文末尾，但必须是独立章节。

推荐结构见 [daily-summary-template.md](daily-summary-template.md)。

标准写法：

```markdown
## 短期预测（1-4周）

> **信息充分性检查结果**：
> - 已有基金自身行为数据：是 / 否
> - 已有市场环境数据：是 / 否
> - 已有行业 / 政策证据：是 / 否
> - 已有反向证据或风险证据：是 / 否
> - 已完成横向比较：是 / 否
> **因此本轮只能给出何种置信度预测。**

| 基金 | 预测方向 | 预计区间 | 依据摘要 |
|------|---------|---------|---------|
| 嘉实新能源新材料A(003984) | ⬇️ 震荡偏弱 | 3.20–3.40 | 支持证据 + 风险边界 |
| 易方达科创50联接C(011609) | ⬆️ 偏强 | 1.26–1.35 | 支持证据 + 风险边界 |
```

要求：

1. 每条预测必须有明确方向。
2. 每条预测必须有预计净值区间。
3. 每条预测必须有依据摘要，且至少包含支持证据与风险/边界之一。
4. 预测日期默认取该 summary 的日期，不要求在表内重复单独增加 `预测日期` 列。

## 规则二：`历史预测验证` 只能来源于历史 `短期预测`

生成当日日报时，必须先读取近期 summary 的 `短期预测` 章节，再填写 `历史预测验证`。

禁止行为：

1. 从观察项、关注要点、操作建议反向推断历史预测。
2. 根据当前结论倒填历史预测内容。

如果近期 summary 都没有 `短期预测` 章节，必须如实说明“无可验证的预测”。

标准写法：

```markdown
## 历史预测验证

> ⚠️ 近期报告均无 `短期预测` 章节，本轮无可验证的预测。

**本轮准确率**：N/A（无历史预测可供验证）
```

## 规则三：验证结果填写规范

当存在历史 `短期预测` 可验证时：

```markdown
## 历史预测验证

| 编号 | 预测日期 | 基金 | 原预测方向 | 原预测区间 | 当前验证净值 | 是否准确 |
|------|---------|------|---------|---------|---------|--------|
| S1 | 2026-05-09 | 嘉实新能源新材料A(003984) | ⬇️ 震荡偏弱 | 3.20–3.40 | 3.28 | ✅ 准确 |
| S2 | 2026-05-09 | 易方达科创50联接C(011609) | ⬆️ 偏强 | 1.26–1.35 | 1.21 | ❌ 错误 |

**本轮准确率**：1/2 = 50%
**说明**：S2 方向偏强但实际走弱，下轮应收窄置信度或下调区间。
```

准确性判断标准：

- ✅ 准确：净值落在预测区间内，且方向吻合。
- ❌ 错误：净值超出预测区间，或方向相反。
- ⚠️ 部分准确：净值超出区间，但方向正确且偏差较小。

## 操作流程

1. 从 `投资新闻归档/index.json` 获取近 1-4 周 summary 列表。
2. 逐一定位 `## 短期预测（1-4周）` 章节。
3. 提取涉及当前持仓的预测条目。
4. 用本轮前一交易日官方净值进行验证。
5. 把验证结论写入 `历史预测验证`。
6. 再写本轮 `短期预测`。

## 提取示例

```python
import re

def extract_prediction(content, fund_code, summary_date):
    pattern = rf"\|\s*([^|]*{re.escape(fund_code)}[^|]*)\|\s*([^|]+)\|\s*([^|]+)\|\s*([^|]+)\s*\|"
    match = re.search(pattern, content)
    if not match:
        return None
    return {
        "fund": match.group(1).strip(),
        "direction": match.group(2).strip(),
        "range": match.group(3).strip(),
        "reason": match.group(4).strip(),
        "predict_date": summary_date,
    }
```

## 与日报模板的对应关系

当前推荐顺序：

1. `历史预测验证`
2. `复盘与风险雷达`
3. `短期预测（1-4周）`

先验证，再复盘，再给新预测，避免自我循环验证。
